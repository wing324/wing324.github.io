Redis的数据结构

Posted on 2015-12-06   |   In Redis   |  

Redis的数据结构多种多样，千姿百态~

String字符串

  1. set
    设置键值

  2. get
    获得键值

  3. expire
    设置键值过期时间

  4. ttl
    显示键值是否过期及过期时间（正数：过期秒数，-1：永不过期，-2：已经过期）

  5. INCR
    让键值自增

  6. DEL
    删除键值

list列表

  1. RPUSH
    在列表的右侧添加值

  2. LPUSH
    在列表的左侧添加值

  3. LRANGE
    从左开始按照对应的范围去取列表值，-1表示一直到最后一个列表值，注意Redis的列表从0开始标记元素

  4. LLEN
    显示列表的长度

  5. LPOP
    删除列表左起第一个元素

  6. RPOP
    删除列表右起第一个元素

SET集合

  1. SADD
    在集合中添加值

  2. SREM
    删除集合中的指定的值

  3. SISMEMBER
    测试某个值是否在集合中，1表示存在，0表示不存在

  4. SMEMBERS
    显示当前集合的所有值

  5. SUNION
    显示所有集合的合并值

  6. SINTER
    显示两个集合中的交集，即共同元素

  7. SINTERSTORE
    将两个集合中的交集，存入到SINTERSTORE后的第一个参数中

SORTED SET有序的集合

  1. ZADD
    在有序的集合中添加元素

  2. ZRANGE
    查看有序集合的元素

  3. ZCOUNT
    显示有序集合在指定的排名中的数量

  4. ZREVRANK
    显示指定有序集合元素的对应等级

Hashes哈希

  1. HSET
    在哈希中添加元素

  2. HGETALL
    查看哈希中的所有元素

  3. HMSET
    对于哈希类型，一次设置多个值

  4. HGET
    只获取哈希类型的部分指定元素

  5. HINCRBY
    对哈希值自增

  6. HDEL
    删除对应的哈希值
